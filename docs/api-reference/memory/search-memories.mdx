---
title: 'Search Memories'
openapi: post /v2/memories/search/
---

The v2 search API is powerful and flexible, allowing for more precise memory retrieval. It supports complex logical operations (AND, OR, NOT) and comparison operators for advanced filtering capabilities. The comparison operators include:
- `in`: Matches any of the values specified
- `gte`: Greater than or equal to
- `lte`: Less than or equal to
- `gt`: Greater than
- `lt`: Less than
- `ne`: Not equal to
- `icontains`: Case-insensitive containment check
- `*`: Wildcard character that matches everything

<CodeGroup>
```typescript Platform API Example
const relatedMemories = await client.search(
  "What are Alice's hobbies?",
  {
    filters: {
      OR: [
        { userId: "alice" },
        { agentId: { in: ["travel-agent", "sports-agent"] } }
      ]
    }
  }
);
```

<CodeGroup>
```typescript Wildcard Example
// Using wildcard to match all run_ids for a specific user
const allMemories = await client.search(
  "What are Alice's hobbies?",
  {
    filters: {
      AND: [
        { userId: "alice" },
        { runId: "*" }
      ]
    }
  }
);
```
</CodeGroup>

<CodeGroup>
```typescript Categories Filter Examples
// Example 1: Using 'contains' for partial matching
const financeMemories = await client.search(
  "What are my financial goals?",
  {
    filters: {
      AND: [
        { userId: "alice" },
        { categories: { contains: "finance" } }
      ]
    }
  }
);

// Example 2: Using 'in' for exact matching
const personalMemories = await client.search(
  "What personal information do you have?",
  {
    filters: {
      AND: [
        { userId: "alice" },
        { categories: { in: ["personal_information"] } }
      ]
    }
  }
);
```
</CodeGroup>
